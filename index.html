Lockdown is a set of scripts, lists and configuration files used to quickly implement granular intelligent automated firewall security at the host level using iptables, ipset and fail2ban. In it's default configuration it is very good at defending web servers running Apache or Nginx and supports ModSec!

Lockdown is used to quickly and easily provide OS-level protection against a wide range of attacks which can be automatically detected by reading log files. It is particularly effective out of the box against brute force and DDOS attacks if your web server is properly configured.

It is designed to be used in automated configurations using Nginx such as Docker or AWS services such as Elastic Container Service or Elastic Beanstalk, but is also well-suited and in use on dedicated servers running cPanel and LAMP stacks.
